## Chapters Overview

### Chapter 1: Basics
Introduction to JavaScript setup, verification, and basic syntax required for Playwright automation.

### Chapter 2: JavaScript Concepts
Core JavaScript concepts that form the foundation for writing robust Playwright test scripts.

### Chapter 3: Identifiers & Literals
- **Identifier Rules**: Valid and invalid naming conventions, different cases (camelCase, PascalCase, snake_case, SCREAMING_SNAKE_CASE, kebab-case), and Hungarian Notation.
- **Comments**: Single-line and multi-line comment usage in JavaScript.

---

## Naming Conventions Covered

| Case | Example | Usage |
|------|---------|-------|
| camelCase | `firstName`, `totalAmount` | Variables, functions |
| PascalCase | `UserProfile`, `PaymentGateway` | Classes, constructors |
| snake_case | `first_name`, `total_amount` | Variables, DB fields |
| SCREAMING_SNAKE_CASE | `MAX_SIZE`, `API_KEY` | Constants |
| kebab-case | `first-name`, `user-data` | File names, CSS classes (not valid for JS identifiers) |
| Hungarian Notation | `sName`, `nCount`, `bIsActive` | Older style (not recommended in modern JS) |
